### Issue Found (Not in Original Plan Scope)

**Stale legalized regions after `eliminate_loops()`**: The `prepare_pipeline()` function at `legalization.py:436-437` has an idempotency early-return — if legalized regions already exist in the program, it returns them without re-validating. When `generate_pytorch_code()` is called for sorting (which legalizes the regions), then `eliminate_loops()` collapses axes in-place, and then `generate_pytorch_code()` or `estimate_program()` is called again, the stale legalized regions with collapsed axes persist. The new strict codegen error at `codegen.py:360-369` then raises `ValueError` instead of falling back gracefully. This affects `example_gemm_ir.py:352-430` and `tests/test_search_gemm.py:70-77` where the pattern `sort(key=generate_pytorch_code) → eliminate_loops → generate_pytorch_code` occurs.
